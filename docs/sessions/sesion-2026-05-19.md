## 1. OBJETIVO DE LA SESIÓN

Resolver los bugs críticos del flujo completo de análisis de lotes: evidencias, catálogos, creación de análisis y tipado de respuestas. La meta era que un usuario pudiera: crear un lote → crear una muestra → crear un análisis → ver evidencias, todo sin errores 4xx/5xx.

## 3. DECISIONES TÉCNICAS TOMADAS

### Decisión 1: `evidencia` no tiene `lote_id` en DB
- **Qué**: La tabla `evidencia` solo tiene `muestra_id`. La relación con lote es indirecta: `evidencia → muestra → lote`.
- **Por qué**: El modelo SQLAlchemy tenía `lote_id` pero la DB real no. No se puede agregar sin migración que rompa datos existentes.
- **Tradeoff**: `lote_id` se guarda en `metadatos` como fallback para evidencias sin muestra asociada.

### Decisión 2: `to_response_dict` → async
- **Qué**: El método síncrono causaba `MissingGreenlet` al hacer lazy-load de relaciones.
- **Por qué**: SQLAlchemy async no permite I/O fuera de un contexto async. El método iteraba `analisis.resultados` sin eager-load.
- **Tradeoff**: Todos los callers deben usar `await`. Se agregaron queries explícitas con fallback si las relaciones ya están cargadas.

### Decisión 3: `lote.status = "analisis_completo"` (string)
- **Qué**: Se usa string en vez del enum Python `LoteStatus.ANALISIS_COMPLETO`.
- **Por qué**: La columna usa `SQLEnum(create_type=False)` que espera el valor string, no el objeto enum.
- **Tradeoff**: Pierde type-safety en el código, pero funciona con el enum de DB existente.

### Decisión 4: Catálogos sin paginación
- **Qué**: Endpoints de variedades, subvariedades, temporadas y parámetros retornan todo de una vez.
- **Por qué**: Son catálogos pequeños (< 50 registros). El frontend usa `staleTime: Infinity`.
- **Tradeoff**: Si los catálogos crecen mucho, habrá que agregar paginación.

### Decisión 5: Documentos - bucket público con whitelist MIME estricta
- **Qué**: Bucket `documentos` es público (como `evidencias`). Solo se permiten PDF, Word, Excel e imágenes. Tamaño máximo 20 MB.
- **Por qué**: Los compradores deben poder descargar documentos desde la ficha pública sin autenticación. La whitelist previene subida de ejecutables o archivos maliciosos.
- **Tradeoff**: El control de acceso se maneja a nivel de DB (solo se registran documentos de lotes que el usuario puede ver). Si un archivo es público pero su URL no está en la DB, es inaccesible de facto.
- **MIME whitelist**: `application/pdf`, `application/msword`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`, `application/vnd.ms-excel`, `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`, `image/*`
- **Soft delete**: `status = 'eliminado'` en vez de borrar físicamente, para auditoría.
